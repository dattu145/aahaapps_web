We have now added Automated Deployment (CI/CD). This means when you push code to GitHub, it will automatically update your live site.

1. Prepare GitHub Secrets

To let GitHub update your cPanel files, it needs permission. We give it permission using "Secrets".

  1. Go to your GitHub Repository page.
  2. Click Settings (Top right tab).
  3. In the left sidebar, scroll down to Secrets and variables -> Click Actions.
  4. Click New repository secret (Green button).
  5. Add these 3 secrets:
  • Name: FTP_SERVER
    Value: ftp.aahaapps.com (or your cPanel IP address / domain)
  • Name: FTP_USERNAME
    Value: deployer@aahaapps.com (See Step 1.1 below to create this!)
  • Name: FTP_PASSWORD
    Value: (The password you set for this FTP user)

Step 1.1: Create the FTP Account

  1. In cPanel, go to FTP Accounts.
  2. Login: deployer
  3. Password: (Generate a strong one).
  4. Directory: /home/mpqptrjm/
    CRITICAL: Delete the default text and make sure it is exactly your home root (or blank). The user needs access to aaha_main_backend_node.
  5. Click Create FTP Account.

2. Push Code to GitHub

I have already created the workflow file for you at .github/workflows/deploy.yml.

Now, simply commit and push your code:

git add .
git commit -m "Setup CI/CD deployment to cPanel"
git push origin main

1. Go to the Actions tab in your GitHub repo.

2. You will see a new workflow running called Deploy to cPanel.

3. If it turns Green âœ…, your site is updated!

3. Future Updates

From now on, whenever you want to update the site:

  1. Make changes on your computer.
  2. Testing locally? Run npm run dev.
  3. Ready to go live? Just run:
git add .
git commit -m "Description of changes"
git push origin main

GitHub will handle the build, file moving, and upload for you automatically!
